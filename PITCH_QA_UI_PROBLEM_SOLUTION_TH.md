# Pitch Q&A: Problem / Solution / UI

## Q: ปัญหาที่แท้จริงของงานนี้คืออะไร?

ปัญหาจริงไม่ใช่แค่ราคาน้ำมันผันผวน แต่คือผลกระทบต่อระบบพลังงานไทยมาแบบหน่วงเวลา ทำให้ผู้วางแผนเห็นสัญญาณช้าเกินไป

## Q: แล้ว SHOCKWAVE แก้ปัญหานี้ยังไง?

SHOCKWAVE เปลี่ยน shock ระดับโลกให้เป็น scenario ที่ผู้ใช้กดทดลองได้ แล้วคำนวณผลกระทบล่วงหน้าต่อ import volume และ diesel sales พร้อมชี้ critical month ที่ควรเฝ้าระวัง

## Q: ความใหม่ของ solution คืออะไร?

ความใหม่คือเราไม่ได้มองแค่ราคาหรือ trend แต่เรามอง hidden lag ระหว่าง global shock กับ local impact เป็นสัญญาณที่มีคุณค่าในตัวเอง และนำมันมาแสดงผ่าน dashboard ที่ใช้งานได้จริง

## Q: ทำไมต้องเป็น dashboard ไม่ใช่แค่รายงานหรือกราฟใน notebook?

เพราะโจทย์นี้ต้องการเครื่องมือที่ non-technical user ใช้ตีความได้เร็ว ผู้ใช้ต้องตั้ง scenario เอง เห็นผลกระทบทันที และอ่านช่วงเวลาเสี่ยงได้จาก critical month โดยไม่ต้องเปิดโค้ด

## Q: หน้าจอ UI นี้ควรดูยังไง?

ดูเป็น 3 ส่วน:

- ซ้าย: ตั้ง scenario
- กลาง: ดูผลสรุปและกราฟ
- ล่าง: ดูตัวเลขรายเดือน

ดังนั้น flow ของผู้ใช้คือ ตั้งค่า กดรัน อ่านผลสรุป แล้วค่อยลงรายละเอียด

## Q: ทำไม UI นี้ถึงสำคัญกับการ pitch?

เพราะเกณฑ์กรรมการให้คะแนน Interactive Web Dashboard ชัดเจน และ UI นี้ช่วยให้คนดูเข้าใจ solution ภายในไม่กี่วินาที ไม่ต้องตีความโมเดลเอง

## Q: ถ้ากรรมการถามว่า dashboard นี้มีประโยชน์จริงยังไง?

ประโยชน์ของมันคือทำให้ risk signal ถูกสื่อสารได้ทันที ผู้ใช้เห็นว่าผลกระทบเริ่มเมื่อไร หนักช่วงไหน และสามารถใช้ scenario planning ได้ทันทีจากหน้าจอเดียว

## Q: ถ้ากรรมการถามว่า solution นี้ต่างจาก dashboard ทั่วไปยังไง?

ต่างตรงที่มันไม่ใช่ dashboard สรุปอดีตอย่างเดียว แต่มันเป็น scenario simulator ที่รับ input จากผู้ใช้ แล้วสร้าง forecast และ critical month จากโมเดลวิเคราะห์จริง

## Q: ถ้ากรรมการถามว่า UI ที่ดีของงานนี้คืออะไร?

UI ที่ดีของงานนี้คือ UI ที่ทำให้คนดูตอบ 3 คำถามได้ทันที:

- shock เท่าไร
- เริ่มส่งผลเมื่อไร
- หนักแค่ไหน

## Q: ถ้าต้องอธิบายงานทั้งก้อนใน 30 วินาที จะพูดยังไง?

"SHOCKWAVE คือ interactive early-warning dashboard ที่ช่วยให้เราเห็นว่า เมื่อ oil shock เกิดขึ้นในตลาดโลกแล้ว ผลกระทบจะค่อย ๆ ส่งผ่านมายังระบบพลังงานไทยเมื่อไร หนักแค่ไหน และช่วงไหนคือหน้าต่างเสี่ยงที่ควรเฝ้าระวัง"
