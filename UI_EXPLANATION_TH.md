# SHOCKWAVE UI Explanation

## ไฟล์นี้ใช้ทำอะไร

ไฟล์นี้คือคำอธิบายหน้า UI แบบเต็ม

ใช้ได้ 3 แบบ:

- ใช้อ่านเพื่อให้คุณเข้าใจหน้าจอทั้งหมด
- ใช้เป็นบทพูดตอนเดโม
- ใช้ตอบกรรมการเวลาโดนถามว่าแต่ละส่วนของ dashboard ทำอะไร

เป้าหมายคือให้คุณอธิบายได้ครบว่า:

- แต่ละส่วนอยู่ตรงไหน
- ใช้ยังไง
- บอกอะไร
- ดูยังไง

---

# 1. ภาพรวมของหน้าจอ

หน้า UI ของ SHOCKWAVE ถูกออกแบบให้ flow ของผู้ใช้เรียบง่ายมาก

ลำดับการใช้งานคือ:

`ตั้ง scenario -> กดรัน -> อ่านผลสรุป -> ดูกราฟ -> ลงรายละเอียดในตาราง`

ดังนั้นเวลาพูด อย่าพูดเป็นภาษาว่า "ตรงนี้คือ component นี้ ตรงนั้นคือ widget นี้"

ให้พูดเป็นภาษาผู้ใช้แทน:

"ผู้ใช้เริ่มจากตั้ง scenario ทางซ้าย จากนั้นรัน simulation แล้วอ่านผลลัพธ์จากส่วนสรุป กราฟ และตาราง"

---

# 2. โครงหน้าจอทั้งหมด

หน้า UI แบ่งเป็น 3 ชั้นหลัก

## ชั้นที่ 1: ฝั่งซ้าย

คือ `ส่วนควบคุม scenario`

เอาไว้กำหนด input ก่อนส่งไปคำนวณ

## ชั้นที่ 2: ส่วนบนของพื้นที่หลัก

คือ `ส่วนอธิบายบริบท + ผลสรุประดับผู้บริหาร`

เอาไว้บอกว่าระบบนี้กำลังจำลองอะไร และผลสำคัญคืออะไร

## ชั้นที่ 3: ส่วนล่างของพื้นที่หลัก

คือ `กราฟและตาราง`

เอาไว้ดูผลลัพธ์เชิงลึกและเทียบ baseline กับ shocked forecast

---

# 3. Sidebar ด้านซ้าย

## มันคืออะไร

นี่คือ `control panel`

เป็นจุดเริ่มต้นของทุกอย่างในหน้า

## มีอะไรอยู่บ้าง

- ชื่อระบบ `SHOCKWAVE`
- คำอธิบายสั้นของระบบ
- `EIA Brent Price Shock (%)` slider
- `Forecast Horizon (Months)` slider
- ปุ่ม `Run Simulation`
- กล่องข้อความ `Hidden Lag Time`

## ใช้ยังไง

ผู้ใช้ต้องมาที่ฝั่งซ้ายก่อนเสมอ

สิ่งที่ผู้ใช้ทำมี 3 ขั้น:

1. กำหนดขนาด shock
2. กำหนดระยะเวลาที่อยาก forecast
3. กด run simulation

## มันบอกอะไร

ส่วนนี้บอกว่า:

- dashboard นี้ไม่ใช่แค่ดูข้อมูลย้อนหลัง
- มันเป็น `scenario simulator`
- ผู้ใช้ควบคุมสถานการณ์ที่จะทดสอบเองได้

## ดูยังไง

เวลาดู sidebar ให้ตีความว่า:

- ด้านซ้าย = input
- ด้านขวา = output

### ประโยคไว้พูด

"ฝั่งซ้ายเป็นส่วนกำหนด scenario ผู้ใช้สามารถระบุขนาดของ Brent shock และจำนวนเดือนที่ต้องการ forecast ได้ ก่อนให้ระบบคำนวณผลกระทบ"

# 4. Brand Block ด้านบนของ Sidebar

## มันคืออะไร

คือส่วนหัวของระบบ

มี:

- คำว่า `SHOCKWAVE`

- งานนี้คือ early warning system
- โฟกัสคือ delayed impact ต่อระบบพลังงานไทย

## เวลาควรพูดยังไง

"ตั้งแต่หัวหน้าจอ เราวางบริบทไว้ชัดว่า SHOCKWAVE เป็น early warning system ไม่ใช่แค่หน้าจอรายงานราคา แต่เป็นระบบจำลองผลกระทบล่วงหน้า"

# 5. Shock Slider

## ชื่อบนหน้าจอ

`EIA Brent Price Shock (%)`

## มันคืออะไร

เป็นตัวกำหนดขนาด shock ที่เราจะใส่ให้กับ upstream Brent signal

## ใช้ยังไง

เลื่อนซ้ายหรือขวาเพื่อเพิ่มหรือลดเปอร์เซ็นต์ shock

เช่น:

- 10% = shock เบา
- 20% = shock ปานกลาง
- 40% ขึ้นไป = stress scenario ที่รุนแรง

## มันบอกอะไร

ตัว slider นี้แทน `severity ของเหตุการณ์ต้นน้ำ`

ยิ่งเปอร์เซ็นต์มาก:

- shock ต้นน้ำยิ่งแรง
- ผลกระทบปลายน้ำที่ระบบจำลองก็ยิ่งแรงขึ้น

## ดูยังไง

อย่าดู slider นี้แยกเดี่ยว

ต้องดูร่วมกับ:

- summary cards
- import chart
- diesel chart

เพื่อดูว่าเมื่อ input เปลี่ยน output เปลี่ยนอย่างไร

### ประโยคไว้พูด

"ตัว slider นี้แทนความรุนแรงของ shock จากตลาดโลก ยิ่งเราปรับเปอร์เซ็นต์สูง ระบบก็จะจำลองแรงกดดันปลายน้ำที่มากขึ้น"

---

# 6. Horizon Slider

## ชื่อบนหน้าจอ

`Forecast Horizon (Months)`

## มันคืออะไร

เป็นตัวกำหนดว่าอยากมองผลกระทบล่วงหน้ากี่เดือน

## ใช้ยังไง

เลื่อนเพื่อเพิ่มหรือลดจำนวนเดือนที่อยากให้ forecast

## มันบอกอะไร

ตัวนี้สำคัญมาก เพราะจุดขายของงานคือ `lag`

ดังนั้น horizon ทำให้ผู้ใช้เห็นว่า:

- shock ไม่ได้ส่งผลทันที
- ผลกระทบค่อย ๆ ปรากฏในช่วงเดือนถัดไป
- critical month อาจอยู่ห่างจากวันเกิด shock หลายเดือน

## ดูยังไง

ถ้าคุณอยากให้กรรมการเห็น hidden lag ชัด

ให้ตั้ง horizon ยาวพอ เช่น 12 เดือนขึ้นไป  
เพื่อให้เห็นการแยกของ baseline กับ shocked path ชัดเจน

### ประโยคไว้พูด

"ตัว forecast horizon ช่วยให้เราไม่มองแค่ผลระยะสั้น แต่เห็นว่าหลังเกิด shock แล้ว ระบบพลังงานไทยจะค่อย ๆ ตอบสนองในอีกกี่เดือนข้างหน้า"

---

# 7. Run Simulation Button

## มันคืออะไร

คือปุ่มสั่งให้ frontend ส่ง scenario ไปที่ backend

## ใช้ยังไง

หลังตั้งค่าเสร็จ ให้กดปุ่มนี้หนึ่งครั้ง

ระบบจะ:

1. ส่ง request ไปยัง API
2. backend คำนวณผลจำลอง
3. frontend อัปเดต summary, charts, และ table

## มันบอกอะไร

ปุ่มนี้คือจุดที่ยืนยันว่า dashboard นี้ `interactive จริง`

ไม่ใช่ภาพ mockup

## ดูยังไง

เวลาคุณกด:

- ปุ่มจะเปลี่ยนสถานะ
- status banner จะอัปเดต
- กราฟและตารางจะเปลี่ยนตาม input ใหม่

### ประโยคไว้พูด

"เมื่อกด Run Simulation ระบบจะคำนวณ scenario ใหม่ผ่าน backend จริง แล้วอัปเดตผลทั้งหมดบนหน้าจอ"

# 8. Hidden Lag Time Note

กล่องข้อความเล็กใน sidebar ที่อธิบายความคิดหลักของงาน

มันบอกผู้ใช้ตั้งแต่แรกว่า:

"กล่องนี้ย้ำแกนของงานเลยว่า oil shock ไม่ได้ส่งผลทันที และคุณค่าของระบบคือการมองเห็นช่วงหน่วงเวลานั้นก่อน"

# 10. Core Signal Box

## ชื่อในหน้าจอ

`Core Signal`

พร้อมข้อความลักษณะ:

`Brent Shock -> Lagged DOEB Response`

## มันคืออะไร

นี่คือหัวใจของการเล่าเรื่องทั้งหมดใน dashboard

## มันบอกอะไร

มันย้ำให้กรรมการเข้าใจง่ายที่สุดว่า:

- input หลักคือ Brent shock
- output สำคัญคือ delayed response ในข้อมูลไทย

## ดูยังไง

อย่าอ่านกล่องนี้เป็นแค่ decoration

ให้ใช้มันเป็นประโยคสรุปทั้งระบบใน 1 บรรทัด

### ประโยคไว้พูด

"กล่องนี้คือประโยคสรุปของทั้งระบบเลย คือ Brent shock จะไม่สะท้อนทันที แต่จะถูกส่งผ่านไปยัง DOEB response แบบมี lag"

# 12. Summary Metrics 3 ใบ

นี่คือส่วนที่สำคัญมากสำหรับการ pitch

เพราะเป็นส่วนที่คนดูเข้าใจเร็วที่สุด

มี 3 ใบ:

- `Max Drop in Import Volume`
- `Critical Month (Lag Time)`
- `Shock Severity`

---

# 13. Metric 1: Max Drop in Import Volume

## มันคืออะไร

ตัวเลขที่สรุปว่าการนำเข้าน้ำมันดิบได้รับผลกระทบหนักสุดแค่ไหน

## มันบอกอะไร

มันตอบคำถามว่า:

`ผลกระทบสูงสุดต่อ import volume มีขนาดเท่าไร`

## ดูยังไง

ดูตัวเลขหลักก่อน  
แล้วดู subtext ใต้ตัวเลขว่าผูกกับ:

- เดือนที่เกิด
- หรือวิธีคำนวณจาก summary

## ใช้พูดยังไง

"การ์ดใบนี้สรุปผลกระทบสูงสุดต่อ import volume ให้คนดูเข้าใจทันที โดยไม่ต้องอ่านทั้งกราฟก่อน"

---

# 14. Metric 2: Critical Month (Lag Time)

## มันคืออะไร

ตัวเลขที่สำคัญที่สุดในงานนี้

## มันบอกอะไร

มันตอบคำถามว่า:

`เดือนไหนคือช่วงเวลาที่ downstream stress เด่นที่สุด`

นี่คือสิ่งที่เชื่อม dashboard เข้ากับคำว่า early warning

## ดูยังไง

ให้ตีความว่า critical month คือ:

- หน้าต่างเสี่ยง
- ช่วงเวลาที่ควรเฝ้าระวัง
- ช่วงที่ baseline กับ shocked path แยกชัดที่สุดในเชิงผลกระทบ

## ใช้พูดยังไง

"การ์ดนี้คือหัวใจของ narrative ทั้งระบบ เพราะมันแปล hidden lag ออกมาเป็นเดือนที่ผู้ใช้สามารถนำไปตีความต่อเชิงการเฝ้าระวังได้"

---

# 15. Metric 3: Shock Severity

## มันคืออะไร

ตัวเลขที่ย้ำว่า scenario ปัจจุบันรุนแรงเท่าไร

## มันบอกอะไร

มันช่วยให้คนดูไม่ลืมว่า output ทั้งหมดเกิดจาก input เท่าไร

## ดูยังไง

เวลาเปลี่ยน slider ค่าใบนี้จะเปลี่ยนตาม

มันช่วยผูก:

- input scenario
- กับผลลัพธ์ทั้งหมดในหน้า

### ใช้พูดยังไง

"การ์ดใบนี้ช่วยย้ำว่าผลทั้งหมดบนหน้าจอผูกกับระดับ shock ที่ผู้ใช้กำหนดอยู่ตอนนี้"

---

# 17. Import Volume Chart

## มันคืออะไร

กราฟแรกของผลลัพธ์

ใช้เปรียบเทียบ:

- baseline import volume
- shocked import volume

## มันบอกอะไร

มันตอบว่า:

- ผลกระทบต่อการนำเข้าน้ำมันดิบเริ่มเมื่อไร
- ผลต่างมากที่สุดช่วงไหน
- เส้น shocked หลุดจาก baseline แค่ไหน

## ดูยังไง

ให้ดู 3 อย่าง:

1. เส้นเริ่มแยกจากกันเมื่อไร
2. ช่องว่างระหว่างเส้นกว้างสุดเมื่อไร
3. ทิศทางของ shocked line ต่ำกว่า baseline แค่ไหน

## วิธีตีความ

ถ้าเส้นเริ่มแยกชัดในช่วงหลัง  
นั่นคือ evidence ของ hidden lag

### ประโยคไว้พูด

"กราฟนี้ทำให้เราเห็นผลกระทบต่อ import volume แบบ visual ว่า shock ต้นน้ำไม่ได้กระทบทันที แต่ค่อย ๆ ทำให้เส้น shocked path แยกออกจาก baseline"

# 18. Diesel Sales Chart

## มันคืออะไร

กราฟที่สองของผลลัพธ์

ใช้เปรียบเทียบ:

- baseline diesel sales
- shocked diesel sales

## มันบอกอะไร

กราฟนี้สะท้อนฝั่ง downstream demand response

ช่วยให้เห็นว่า:

- diesel sales ตอบสนองต่อ shock หรือไม่
- ตอบสนองเร็วหรือช้ากว่า import volume
- ความรุนแรงของผลกระทบปลายน้ำเป็นอย่างไร

## ดูยังไง

ให้ดูคู่กับ import chart เสมอ

เพราะ narrative ที่ดีคือ:

- import volume เป็นผลกระทบอีกมิติหนึ่ง
- diesel sales เป็น downstream response อีกมิติหนึ่ง

### ประโยคไว้พูด

"กราฟนี้ช่วยให้เราเห็นว่าแรงกดดันจาก shock ไม่ได้สะท้อนแค่ใน import volume แต่ยังไหลต่อไปยังฝั่ง diesel demand ด้วย"

# 19. ทำไมต้องมี 2 กราฟ

เพราะถ้ามีแค่กราฟเดียว  
คนดูจะเห็นแค่ผลกระทบในมิติเดียว

แต่การมีทั้ง import และ diesel ทำให้คุณเล่าได้ว่า:

- shock เดียวกันส่งผลหลายส่วนของระบบ
- การตอบสนองของแต่ละตัวแปรไม่จำเป็นต้องเหมือนกัน
- hidden lag อาจแสดงตัวต่างกันในแต่ละมิติ

### ประโยคไว้พูด

"เราไม่ได้ดูแค่ตัวแปรเดียว เพราะการส่งผ่านของ shock ในระบบจริงมักไม่เกิดในมิติเดียว"
